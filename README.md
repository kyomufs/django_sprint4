Учебный проект "Блогикум".
Выполнил: Kyomu (Матвей Самодуров), 2026.

## Запуск проекта

1. **Клонировать репозиторий** (если еще не сделано):
   ```bash
   git clone <repository-url>
   cd django_sprint4
   ```

2. **Создать и активировать виртуальное окружение**:
   ```bash
   python3 -m venv venv
   source venv/bin/activate   # на Windows: venv\Scripts\activate
   ```

3. **Установить зависимости**:
   ```bash
   pip install -r requirements.txt
   ```

4. **Выполнить миграции базы данных**:
   ```bash
   cd blogicum
   python manage.py migrate
   ```
5. **Запустить сервер разработки**:
   ```bash
   python manage.py runserver
   ```
   Сервер будет доступен по адресу http://127.0.0.1:8000/

6. **Остановить сервер** — нажать `Ctrl+C` в терминале.

## Примечания

- При работе с проектом убедитесь, что используете Python 3.11 или выше (в оригинале использовалось 3.11.15).
- Если возникают ошибки с установкой Pillow, убедитесь, что установлены системные зависимости для обработки изображений (libjpeg, zlib и т.д.).
- Для работы с админ-панелью создайте суперпользователя:
  ```bash
  python manage.py createsuperuser
  ```
  Затем войдите по адресу http://127.0.0.1:8000/admin/

## Структура проекта

- `blogicum/` — основная директория проекта Django.
- `blog/` — приложение с моделями постов, категорий, мест и т.д.
- `pages/` — приложение для статических страниц (about, rules).
- `static_files/` — статические файлы (CSS, JS, изображения).
- `templates/` — HTML-шаблоны.
- `requirements.txt` — список зависимостей Python.

## Стек проекта «Блогикум»:

- Язык: Python (в оригинале использовалось 3.11.15, но работает и с текущими версиями 3.12+)
- Фреймворк: Django 3.2.16 (указано в requirements.txt)
- База данных: SQLite (по‑умолчанию в Django, файл db.json в корне проекта)
- Фронтенд:
- HTML‑шаблоны (директория templates/)
- CSS‑фреймворк Bootstrap 5 через пакет django-bootstrap5
- Статические файлы (директория static_files/)
- Дополнительные библиотеки:
- Pillow – работа с изображениями (ImageField в модели Post)
- Faker – генерация тестовых данных
- mixer – фикстуры для тестирования
- pytest, pytest-django, flake8 и др. – инструменты тестирования и качества кода
- python-dateutil, pytz – работа с датами и временем
- asgiref – асинхронная поддержка (встроена в Django)
